# 个人网站需求说明

> 记录日期：2026-07-10  
> 状态：需求整理；不代表已完成技术栈或模板选型。

## 已确认需求

### 网站定位

- 网站是**个人网站**，而不是单纯的博客分享站。
- 个人信息、项目展示和笔记分享是并列的核心内容。
- 首页应首先表达个人身份、方向和代表性工作，而不是以文章列表作为唯一主体。

### 内容范围

1. **个人信息**
   - 自我介绍与个人定位。
   - 教育/经历、技术方向、技能和联系方式。
   - 可包含 GitHub、邮箱及其他必要的社交链接。

2. **项目展示**
   - 介绍个人项目。
   - 每个项目应能表达问题背景、使用技术、个人工作、成果或实验结果，以及代码/相关链接。
   - 首页需要能够展示少量精选项目。

3. **笔记分享**
   - 发布和沉淀技术笔记。
   - 内容可能涉及 CUDA、Triton、LLM 推理、AI Infra、实验复盘和论文阅读。
   - 笔记应适合 Markdown 写作，并良好支持代码、图片、公式或图表等技术内容。

### 语言与排版

- 网站主要面向中文读者，界面和内容以**简体中文**为主。
- 模板需要对中文有良好适配，而不仅是可以把英文文案手动翻译成中文。
- 应关注中英文混排、数字、代码块、公式、目录、日期和搜索等场景。

### 审美方向

- 审美倾向：**简约、精致、克制**。
- 希望有清晰的信息层级、充分留白和较好的阅读体验。
- 不希望站点呈现为过度装饰的博客、二次元风格主题或营销落地页。
- 动画、渐变、卡片和装饰元素应服务信息表达，不应喧宾夺主。

### 技术与维护偏好

- 可以使用成熟社区模板作为基础，而非必须从零构建。
- 希望模板/技术栈具备：
  - 较好的社区支持；
  - 持续且相对稳定的维护；
  - 不低的使用规模；
  - 清晰的更新、部署和排障资料；
  - 适合长期维护与持续添加内容。
- 希望静态部署到 GitHub Pages，仓库名称为 `yiwen-cai.github.io`。
- 需要控制上游升级和内容迁移成本，避免未来被模板私有格式深度绑定。

## 目标信息架构

下列结构表达已确认的内容边界；具体 URL、命名和布局尚可调整：

```text
首页
├── 个人定位 / 简介
├── 精选项目
├── 最新笔记
└── 个人经历与联系入口

项目
├── 项目列表
└── 项目详情

笔记
├── 主题分类
└── 单篇笔记

关于
└── 个人信息、经历、技能和联系方式
```
